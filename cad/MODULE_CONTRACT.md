# Clements 49 — Module Contract

Canonical interface every component file must expose. The assembly and all
component authors depend on these exact module names + argument lists so the
shared world frame, the string schedule, and the material switch stay
consistent across files. Generated by the foundation task.

## Shared rules (all component files)

- First two lines: `include <../params.scad>` then `include <../strings.scad>`.
  (Component files live under `cad/frame/` and `cad/brass/`, so the parent path
  is `../`.) These are the ONLY config sources; do not hard-code dimensions
  that params/strings already provide.
- World frame: z-up, millimetres, origin = base-footprint centre in x,y and
  z=0 at the floor. Bass->treble = +x. Column/player side = -x. y = across-body
  depth. Every frame module renders IN PLACE in this world frame (no local
  re-centring) so files overlay directly.
- Iterate `for (i = indices())` and read `string_*(i)`, `string_board_pos(i)`,
  `string_pin_pos(i)`. NEVER hard-code 47/49 — the count comes from
  `string_count()`. Adding strings is a data edit in `strings.scad` only.
- Material/colour: read the `material` switch indirectly via the params
  accessor helpers (`wall_*()`, `*_thickness()`, `shell_color()`,
  `string_color()`) and the `as_shell()/as_tonewood()/as_carbon()/as_brass()`
  colour-wrap modules. Do not branch on `material` yourself.
- CASTING HARD CONSTRAINT: generate NO molds/cavities this run. Each part is a
  solid `module` so a future offset-shell can wrap it. Leave clean hooks only.
- Apply `brass_pattern_scale()` ONLY when emitting a casting pattern (future);
  never to the as-used brass geometry in the assembly.

## Component module signatures (the contract)

Each frame module takes NO required args (reads params + strings). Optional
named args have defaults so callers may omit them.

### frame/pillar.scad
- `module pillar();`
  Leaning flax tube (or solid wood column) along the `pillar_bottom_mm ->
  pillar_top_mm` axis, length `pillar_length_mm`, OD `pillar_od()`, wall
  `pillar_wall()`. Solid module; future offset-shell hook.

### frame/neck.scad
- `module neck();`
  Harmonic-curve neck following the tuning-pin line (`string_pin_pos(i)` over
  `indices()`), woven-flax super-ellipse (n = `neck_superellipse_n`) section
  tapering `neck_h_end_mm` -> `neck_h_peak_mm` -> `neck_h_end_mm`, width
  `neck_w_mm`. Exposes pin-seat positions via `string_pin_pos(i)`.
- `module neck_spine();`  (optional sub-part) the CF-UD spine solid; rendered
  only when `is_flax`. Caller may union it into `neck()`.

### frame/soundboard.scad
- `module soundboard();`
  Spine-and-wings board on the board string line (`string_board_pos(i)`).
  U-length `board_span_mm`, taper `board_width_bass_mm` -> `board_width_treble_mm`,
  longitudinal crown `board_crown_radius`, total thickness
  `soundboard_thickness()` (sandwich when flax, solid plate when wood). String
  exit eyelets at `string_board_pos(i)`.
- `module soundboard_spine();` (optional) the load-carrying spine strip alone.
- `module soundboard_wing(side);` (optional, side = -1|+1) one radiating wing.

### frame/soundbox.scad  (the shape to PERFECT)
- `module soundbox();`
  Lofted leaning shell from `soundbox_section_count()` control sections via
  `soundbox_section_eff(i)` (already carries the acoustic tunables:
  `body_*_profile`, `body_length_scale`, `bass_corner_bulge`,
  `cavity_fill_factor`). Wall `wall_soundbox()`. Back crown
  `back_curvature_radius`, liveness `back_liveness`.
- `module soundbox_cavity();` (optional) the INNER lofted volume (air cavity),
  for volume/Helmholtz analysis. Solid positive of the inside; NOT a boolean
  subtraction tool this run.
- `module soundbox_holes();` (optional) `soundhole_count` graded back
  soundholes positioned by `soundhole_axis_offset`, sized by `soundhole_scale`
  and `soundhole_grade_exp`. Marked as a positive-solid hook (do not subtract
  this run unless the caller opts in).

### frame/base.scad
- `module base();`
  Monocoque base/plinth over the footprint (`base_xmin..base_xmax`,
  `base_ymin..base_ymax`, height `base_footprint_z`), wall `wall_base()`.
  CF-reinforced junctions at the pillar foot (`pillar_bottom_mm`) and body
  foot. Houses the (future) pedal mechanism — leave interior as a clean hook.

### brass/brass.scad
- `module tuning_pins();`   one cast pin per string at `string_pin_pos(i)`.
- `module sharping_discs();` double-action fork/disc pair per string near
  `string_pin_pos(i)` / the neck.
- `module action_plates();`  neck action-plate set along the pin line.
- `module pedals();`         7 pedals + linkage at the base.
- `module brass(parts = "all");`  convenience aggregator; `parts` is a string
  selecting `"pins"|"discs"|"plates"|"pedals"|"all"`. All brass modules render
  at nominal (un-shrunk) size; pattern scaling is a future, opt-in step.

## Assembly (future, not built this run)
- `assembly.scad` will `use` each component and call `pillar(); neck();
  soundboard(); soundbox(); base(); brass();` with no args, relying on the
  shared world frame for registration.
